# PCB – Linee guida frontend

## 1. Obiettivo UI

Interfaccia:
- istituzionale
- chiara
- sobria
- moderna
- orientata a operatori interni

## 2. Stack frontend

- Next.js
- React
- Tailwind CSS
- React Query
- OpenLayers

## 3. Sezioni principali dell'app

- Dashboard
- Ricerca globale
- Scheda soggetto
- Scheda particella
- Mappa GIS
- Ingestion monitor
- Audit / log
- Admin settings

## 4. Principi UX

### P-01
La scheda soggetto è centrale.

### P-02
L'utente deve poter passare facilmente da:
- soggetto
- particella
- documento
- evento
- mappa

### P-03
L'interfaccia non deve imitare i portali esterni. Deve essere superiore in chiarezza.

### P-04
Le schermate devono essere stabili e comprensibili, non decorative.

## 5. Componenti chiave

- search bar globale
- risultati raggruppati
- card identità soggetto
- timeline storico
- tab documenti
- tab particelle
- tab GIS
- tab sorgenti collegate

## 6. Mappe

La mappa GIS deve consentire:
- visualizzazione layer
- selezione feature
- link a soggetti e particelle
- filtri tematici

## 7. Design system

- colori sobri
- stato attivo/inattivo chiaro
- tipografia leggibile
- griglie e spacing regolari
